# 參與開發

感謝協助完善《夜行列車：守夜協定》。請先建立 issue 說明玩法、數值或視覺目的，再提交小而可驗證的變更。

## 開發流程

1. Fork 儲存庫並建立功能分支。
2. 執行 `npm ci`。
3. 修改資料時維持 `src/game/` 的權威服務層，不從 UI 直接改寫資源。
4. 新增 shipping 圖像時，同步更新 `docs/ASSET_MANIFEST.md`、runtime artKey 與 `tests/assets.test.ts`。
5. 執行 `npm run check`。
6. PR 說明需包含目的、玩家影響、測試與手機截圖。

## 設計規則

- 玩家是守護 AI；不加入自由移動、搖桿、射擊或角色戰鬥。
- 固定單節車廂與 9:16 手機優先。
- 最小觸控區 48×48 CSS px。
- 資訊不能只靠顏色表達。
- 不得把完整 UI 視覺稿攤平當成互動畫面。
- runtime 不得包含 OpenAI、xAI 或任何第三方服務金鑰。

提交即代表你同意以本專案標示的 AGPL-3.0-or-later 授權程式碼／文件；原創美術則以 CC BY 4.0 提供。
