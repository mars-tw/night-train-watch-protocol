## 0. 實作前必讀

- 16 張畫面皆為 720×1280 PNG，代表 360×640 logical px 的 2× 輸出。
- 圖上的橘色圓形數字 1–5、最底部「版面設計稿・美術可替換・邏輯尺寸 360×640」標籤，以及右下 `SCR-*` 編號都是文件註記，不是遊戲 UI，實作時必須移除。
- PNG 已把背景、UI、註記合成在同一層，不能直接整張貼入遊戲。正確做法是把畫面結構、場景、美術、模組、VFX、UI 分層重建。
- 所有 A/B 稿都不是兩套風格，而是同一視覺系統下的標準狀態與壓力／替代狀態。

### 字體與字級

- 稿內沒有標示確切字型檔名稱；不要宣稱已指定某個商用字型。
- 視覺角色：遊戲標誌、結局大字、事件標題用 CJK 明體／襯線；其餘控制項、資料、卡片用 CJK 黑體／無襯線。
- 頁面標題 28 logical px；區塊標題 20；正文 16；輔助資訊 12。
- 數值採等寬數字或 `font-variant-numeric: tabular-nums`，資源數、倒數、百分比不得跳寬。
- 720×1280 實際輸出時，上述字級乘 2；Web/CSS 仍以 logical px 撰寫。

### 格線、留白與尺寸

- 基礎格線：8 logical px；半格 4 px 僅用於文字與圖示微調。
- 安全區：上 30、下 26 logical px。
- 最小觸控區：44×44 logical px。
- 頁面左右基準邊距：12 logical px；面板內距 12；卡片間距 8；主區塊間距 16。
- 一般卡片／HUD 圓角約 8 logical px；大面板約 12；膠囊標籤使用全圓角。
- 一般描邊 1 logical px；選取、主 CTA、警報面板用 2 logical px。

### 層級與材質

1. 像素美術車廂／窗外場景。
2. 黑色或冷藍半透明遮罩與景深，保證文字對比。
3. `#192329` 高不透明度 UI 面板。
4. 正文、圖示、數值。
5. 暖色選取／CTA、冷色資訊、安全綠、危險紅狀態。
6. 警報、倒數、手勢提示等暫態層。

場景特徵必須遵循原稿：像素美術、暖色室內燈、冷色窗外雪霧、五類車廂（臥室、武器／物資、工坊／情報、溫室、廚房／儲藏）。不得替換為霓虹太空、泛用暗黑卡片或純幾何戰鬥場。

## 3. 共用元件清單

優先建立下列共用元件，不要每頁各寫一套：

1. `AppHeader`：返回、標題、副標、右側資源／速度／暫停。
2. `ResourceBar`：電力／燃料，支援稀缺色與數字上限。
3. `Panel`／`Card`／`Pill`：一般、選取、禁用、鎖定、安全、危險六狀態。
4. `PrimaryAction`：每屏最多一個暖金高亮 CTA。
5. `BottomNav`：中心、路線、科技、圖鑑四項。
6. `BottomSheet`：模組目錄與配置工具；支援拖曳把手與保留背景。
7. `RouteNode`／`TechNode`：以顏色、輪廓、填色、形狀同時傳達狀態，不能只靠顏色。
8. `ChoiceCard`：事件選項同時呈現名稱、成本、已知後果。
9. `StatusMeter`／`RingMeter`：生命／車體／睡眠／倒數。
10. `SaveStatus`：存檔存在、無存檔、自動存檔完成、雲端未連線。

## 4. 互動狀態總表

- `default`：面板底＋冷色細框。
- `selected`：暖金 2px 框，可加暖金圖示底，但不要整頁多個主色 CTA。
- `disabled`：文字、圖示、框線整體降對比；仍保留原因或狀態文字。
- `locked`：降對比＋「需科技／未解鎖／鎖」文字，不只鎖頭圖示。
- `safe/success`：綠色；產出、可放、存檔成功。
- `info/scanned`：冷藍；資料、掃描、天候修正。
- `scarce/resource`：淡黃；電力、燃料與稀缺成本。
- `danger/urgent`：紅色；面板紅框、倒數、風險與不可放置。
- `pressed/focus`：稿中未給具體圖，實作應維持原色系，使用 2px 外框／亮度位移；鍵盤焦點不可移除。
- `reduced motion`：倒數仍保留數字；警報閃爍要能停用，以靜態紅框與文字替代。

## 5. 可直接裁切素材與限制

### 可用於原型

- 01A／01B 的鐵軌符號＋遊戲標誌，可裁後去背；建議最終仍重排文字，避免位圖字縮放。
- 05A 開場事件插圖，約 `x=87,y=238,w=547,h=236`（720×1280 稿座標），是 16 張中最接近可直接使用的乾淨矩形素材。
- 02B 的路線縮圖、06A 的模組 glyph、07 的科技節點符號可快速裁成原型佔位，但正式版應重畫 SVG／Canvas。
- `00_場景美術方向參考圖.png` 中 5 種車廂各 A/B 圖可作構圖與物件清單參考；內含方案標籤、尺寸偏小，不是正式背景資產。

### 不可直接當正式資產

- 03A／03B 車廂場景：已疊 HUD、熱點、角色手勢與編號。
- 04A／04B 路線地圖：應由資料生成，不能用固定截圖。
- 05B 倒數圈、07 科技樹、08A 統計圓環：應由數值即時繪製。
- 所有整頁 PNG：含文件註記、Footer、扁平 UI，不得當遊戲畫面貼圖。

## 6. 驗收基準

- 360×640 logical viewport 首屏必須與對應稿的資訊密度、層級和 A/B 狀態一致。
- 主要 CTA 每屏最多一個，且位於原稿對應位置。
- 上下固定區、Bottom Sheet、地圖／科技主區比例不可任意改動。
- 所有按鈕至少 44×44 logical px；正文與背景需維持可讀對比。
- 場景、UI、文字、角色、VFX 必須能分層替換。
- 截圖比對時先隱藏文件編號圓圈與底部稿件 Footer，再比對畫面。
